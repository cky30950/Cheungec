需要設置 Firebase 才能跨設備同步資料

目前使用示範配置，資料只能在本設備使用。請按照以下步驟設置真實的 Firebase 配置：

🔥 Firebase 設置步驟

步驟 1：創建 Firebase 專案

  1. 前往 Firebase Console
  2. 點擊「建立專案」
  3. 輸入專案名稱（例如：tcm-helper）
  4. 選擇是否啟用 Google Analytics（可選）
  5. 點擊「建立專案」

步驟 2：設置 Firestore 資料庫

  1. 在 Firebase Console 中，點擊左側選單的「Firestore Database」
  2. 點擊「建立資料庫」
  3. 選擇「以測試模式啟動」（30天內可讀寫）
  4. 選擇資料庫位置（建議選擇 asia-east1）
  5. 點擊「完成」

步驟 3：註冊網頁應用程式

  1. 在專案總覽頁面，點擊「</>」圖示（網頁）
  2. 輸入應用程式暱稱（例如：中醫工具）
  3. 不需要勾選「設定 Firebase Hosting」
  4. 點擊「註冊應用程式」

步驟 4：獲取配置資訊

  1. 在「新增 Firebase SDK」頁面，複製 firebaseConfig 物件
  2. 或者到「專案設定」→「您的應用程式」→「SDK 設定和設定」
  3. 複製類似以下的配置：
const firebaseConfig = {
  apiKey: "AIzaSyC...",
  authDomain: "your-project.firebaseapp.com",
  projectId: "your-project",
  storageBucket: "your-project.appspot.com",
  messagingSenderId: "123456789",
  appId: "1:123456789:web:abc123"
};

步驟 5：更新程式碼

  1. 將獲取的配置資訊替換程式碼中的示範配置
  2. 找到程式碼中的 firebaseConfig 部分
  3. 將「您的-API-密鑰」等替換為真實值
  4. 儲存並重新載入頁面

步驟 6：設置安全規則（重要）

  1. 在 Firestore Database 頁面，點擊「規則」標籤
  2. 將規則更新為以下內容（測試用）：
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /{document=**} {
      allow read, write: if true;
    }
  }
}

⚠️ 這是測試規則，生產環境需要更嚴格的安全設置

✅ 驗證設置

完成設置後，您應該看到：

  • 右上角狀態顯示「已連接」（綠色）
  • 可以成功執行手動備份
  • 在其他設備上載入相同資料

🔧 常見問題

  • 連接失敗：檢查配置是否正確複製
  • 權限錯誤：確認 Firestore 規則已更新
  • 找不到專案：確認 projectId 正確
  • 網路錯誤：檢查網路連接和防火牆設置
